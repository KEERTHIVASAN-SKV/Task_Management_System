WHY THIS HAPPENS:
=================

Timeline:
---------
1. First version: script.js used port 8080
2. You changed backend to port 8081
3. I updated script.js to use port 8081
4. BUT your browser still has the OLD script.js cached
5. Browser uses cached version (port 8080) instead of new one (8081)

Solution:
---------
Hard refresh forces browser to download fresh files!

TROUBLESHOOTING:
================

Q: I pressed Ctrl+Shift+R but still see error
A: Try these in order:
   1. Close ALL browser tabs
   2. Close browser completely
   3. Reopen browser
   4. Open index.html
   5. Press Ctrl+Shift+R again

Q: Still not working?
A: Try incognito mode:
   1. Ctrl+Shift+N (Windows) or Cmd+Shift+N (Mac)
   2. Open index.html in incognito window
   3. Should work immediately

Q: How do I know if cache is cleared?
A: Open browser console (F12) and look for:
   "API URL: http://localhost:8081/api/tasks"
   If you see 8081 → Cache cleared!
   If you see 8080 → Cache not cleared yet

Q: Can I just delete the cache files manually?
A: Hard refresh is easier, but you can:
   1. Browser Settings
   2. Privacy & Security
   3. Clear browsing data
   4. Select "Cached images and files"
   5. Clear data


SUCCESS INDICATORS:
===================

✓ Console shows: "API URL: http://localhost:8081/api/tasks"
✓ Page loads without error message
✓ Can click "Add New Task" button
✓ Modal opens when clicking button
✓ Can create and save tasks
✓ Tasks appear in the list


PREVENTION:
===========

To avoid this in the future:

1. Enable "Disable cache" in browser dev tools:
   - Press F12
   - Go to Network tab
   - Check "Disable cache"
   - Keep dev tools open while developing

2. Always use hard refresh when testing changes

3. Test in incognito mode during development
